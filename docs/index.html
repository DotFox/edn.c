Nil

Represents nil, null, or nothing. Written as nil.

nil

Booleans

The boolean values true and false.

true
false

Integers

64-bit signed integers. An integer can have the suffix N to indicate arbitrary precision is desired.

42
-17
+5
0
9999999999999999999N

Floating Point Numbers

64-bit (double) precision floating point numbers. The suffix M indicates exact precision decimal (BigDecimal).

3.14159
-2.5e10
6.022E23
3.14159265358979323846M

Characters

Characters are preceded by a backslash. Special named characters and Unicode escapes are supported.

\a

Strings

Strings are enclosed in double quotes. Standard escape sequences are supported.

Escape Meaning
\\Backslash
\"Double quote
\nNewline
\rCarriage return
\tTab
 
"Hello, World!"
"Line 1\nLine 2"
"She said \"Hi!\""

Symbols

Symbols represent identifiers. They can optionally be namespaced with a / separator.

foo
my-function
clojure.core/map
user/config

Symbol names may contain alphanumeric characters and . * + ! - _ ? $ % & = < >. They must begin with a non-numeric character (and cannot start with : or #).

Keywords

Keywords are like symbols but begin with a colon :. They typically designate themselves and are semantically akin to enumeration values.

:name
:user/email
:status

Lists

A list is an ordered sequence of values enclosed in parentheses.

(1 2 3)
(println "hello")
(:a :b :c)

Vectors

A vector is an ordered sequence of values enclosed in square brackets. Vectors support efficient random access.

[1 2 3]
["a" "b" "c"]
[:mixed 42 true nil]

Maps

A map is a collection of key-value pairs enclosed in curly braces. Keys can be any EDN value (not just strings). Each key should appear at most once.

{:name "Alice" :age 30}
{"string-key" 123}
{[1 2] "composite key"}

Sets

A set is a collection of unique values, written as #{...}.

#{1 2 3}
#{:a :b :c}
#{"apple" "banana" "cherry"}

Tagged Elements

EDN is extensible through tagged literals. A tag is a symbol prefixed with # that indicates the semantic interpretation of the following element.

#inst "2024-01-15T10:30:00.000Z"
#uuid "f81d4fae-7dec-11d0-a765-00a0c91e6bf6"
#myapp/person {:name "Bob" :age 25}

Built-in Tagged Elements

Tag Description Example
#inst An instant in time (RFC-3339 format) #inst "2024-01-15T10:30:00Z"
#uuid A UUID #uuid "f81d4fae-7dec-11d0-a765-00a0c91e6bf6"

User-defined tags must contain a namespace prefix (e.g., #myapp/type). Tags without a prefix are reserved for built-in extensions.

Grammar

Whitespace

Whitespace characters separate elements. Commas are considered whitespace—they're optional and have no semantic meaning.

; These are equivalent:
[1 2 3]
[1, 2, 3]
[1,2,3]

Valid whitespace characters: space, tab (\t), newline (\n), carriage return (\r), and comma (,).

Comments

Comments start with a semicolon and continue to the end of the line.

; This is a comment
{:name "Alice"  ; inline comment
 :age 30}

Discard

The #_ form causes the next element to be read and discarded. Useful for commenting out data.

[1 #_2 3]         ; => [1 3]
{:a 1 #_:b #_2}   ; => {:a 1}

Clojure Extensions

While the base EDN specification covers the core data types, Clojure and some EDN parsers support additional extensions:

Ratio Numbers

Exact rational numbers represented as numerator/denominator pairs.

22/7
-3/4
1/2

Map Namespace Syntax

A shorthand for namespacing all keyword keys in a map.

#:person{:name "Alice" :age 30}
; Equivalent to:
{:person/name "Alice" :person/age 30}

Metadata

Attach metadata to values using the ^ prefix.

^:private my-var
^{:doc "A function"} my-fn

Extended Integer Formats

Hexadecimal, octal, binary, and arbitrary radix numbers.

0xFF      ; hexadecimal (255)
0777      ; octal (511)
2r1010    ; binary (10)
36rZZ     ; base-36 (1295)

Note: These extensions are not part of the core EDN specification. Check your parser's documentation for support.

Equality

EDN defines equality semantics for comparing values:

  • Nil, booleans, strings, characters, symbols: Equal if of the same type with the same representation
  • Numbers: Equal only if of the same magnitude, type, and precision
  • Sequences (lists, vectors): Equal if they have the same count and each corresponding element is equal
  • Sets: Equal if they have the same count and for every element in one set, an equal element is in the other
  • Maps: Equal if they have the same number of entries and for every key/value pair in one map, an equal key is present and mapped to an equal value in the other
  • Tagged elements: Define their own equality semantics
